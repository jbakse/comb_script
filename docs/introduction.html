Document Structure

Comb Script documents are written in YAML [link], a common and human-readable data-file format. Comb Script recognizes predefined keywords to describe regions, shapes, and their properties.

Regions

The core element of a Comb Script document is a Region. A Region defines a positional boundary in the document and can contain child Regions. Plain Regions are primarily organizational and do not correspond to specific output shapes. Specialized Region Types like Rectangles and Ellipses generate output.

Properties

Region properties describe specific aspects of Regions, such as `width` and `height`. Properties are described as a key-value map under the heading `properties:`. 
#very_simple.yaml
#a very simple comb-script
children:
    - region:
        properties:
            width: 200
            height: 150

Nesting and Children

Nesting Regions is a key structural element of Comb Script documents. Regions can contain zero or more child regions. Nested regions are positioned and sized relative to the bounds of their parent. You can also nest regions just to make your documents easier to understand. A region's children are defined in an array under the heading `children:`. 
#nesting_simple.yaml
children:
    #this region is a child of the 'document'
    - region:
        properties:
            width: 300
            height: 300
        #and also has a child of it's own
        children:
            #children are defined in an array
            - region:
                properties:
                    name: only_child
                    height: 100
                    width: 100
                    margin_left: 30

Bounds

Every region has a rectangular bounds. These boundaries can specified directly with the `top`, `left`, `bottom`, and `right` properties. These properties use the coordinate system set by the region's parent. A region's bounds can also be specified relative to the parent's bounds using `margin-top`, `margin-left`, `margin-bottom`, and `margin-right` properties. You can specify the dimensions of the bounds with the `width` and `height` properties. You can mix these properties any way you wish, as long as they don't conflict. 
#bounds_simple.yaml
children:
    - region:
        #this region is 200x200 units
        properties:
            top: -100
            left: -100
            bottom: 100
            right: 100

Registration

Regions can optionally redefine the zero point of the coordinate system with the `registration` property. By default `registration` is set to `parent` which will preserve the location of the origin set by the region's parent. Other values, such as 'top_left' and 'center' move the origin relative to the Region's bounds.

Transformations

You can specify transformations to apply to a region's shape and it's children with the `rotation`, `scale_x`, and `scale_y` properties.

Shapes

The basic Region type is primarily organizational and will not generate a vector shape in the exported SVG. To create a exported shape, use specialized Region-types such as Rectangles and Ellipses.

Boolean Operations

Complex Shapes can be created by combining basic shapes using boolean operations. Use the `boolean` property combine a region's shape witto `add`, `subtract`, or `intersect`.
